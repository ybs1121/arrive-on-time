# 도메인 노트

## 유비쿼터스 언어

### 도착 정책 (ArrivalPolicy)
사용자가 원하는 도착 조건을 표현하는 개념이다.
도착 시간을 엄격하게 지켜야 하는지,
혹은 몇 분 정도 늦어도 괜찮은지를 포함한다.

### 예상 이동 시간 (TravelDuration)
출발지에서 도착지까지 이동하는 데 걸릴 것으로 예상되는 시간이다.
외부 길찾기 API의 결과를 내부 도메인 개념으로 해석한 값이다.

### 여유 시간 (BufferMinutes)
예상 이동 시간 외에 추가로 반영하는 안전 여유 시간이다.
엄격 도착 여부, 이동 시간 길이 등에 따라 달라질 수 있다.

### 추천 출발 결과 (DepartureRecommendation)
사용자의 도착 조건과 예상 이동 시간, 여유 시간을 반영해 계산된 추천 출발 결과이다.

### 추천 엔진 (RecommendationEngine)
도착 정책, 이동 시간, 여유 시간을 조합하여 추천 출발 시간을 계산하는 핵심 도메인 서비스이다.

### 여유 시간 정책 (BufferPolicy)
도착 정책과 이동 시간에 따라 얼마의 여유 시간을 반영할지 결정하는 정책이다.

## 현재 도메인 규칙

### 규칙 1
strict = true 이면
목표 도착 시간이 최종 도착 기준 시간이다.

### 규칙 2
strict = false 이면
목표 도착 시간 + allowedDelayMinutes 가 최종 도착 기준 시간이다.

### 규칙 3
추천 출발 시간은 아래 기준으로 계산한다.

추천 출발 시간 = 최종 도착 기준 시간 - 예상 이동 시간 - 여유 시간

### 규칙 4
엄격 도착 여부와 이동 시간 길이에 따라 여유 시간이 달라질 수 있다.

예시:
- 엄격 도착이면 기본 10분
- 여유 도착이면 기본 3분
- 이동 시간이 60분 이상이면 추가 5분

## 설계 메모
- 외부 API 응답은 내부 도메인 모델로 변환해서 사용한다.
- 도메인 규칙은 컨트롤러가 아니라 도메인 서비스에서 처리한다.
- 추천 계산 로직은 외부 API 호출 없이도 테스트 가능해야 한다.
- 도메인 용어는 프로젝트 전반에서 일관되게 사용한다.
