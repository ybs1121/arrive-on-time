# 프로젝트 개요

## 프로젝트명
ArriveOnTime

## 한 줄 소개
출발지, 도착지, 목표 도착 시간, 도착 엄수 여부, 허용 지연 시간을 입력받아
추천 출발 시간을 계산해주는 서비스입니다.

## 프로젝트 목표
이 프로젝트의 핵심은 단순히 지도 API를 붙이는 것이 아니라,
도착 제약조건을 해석하여 추천 출발 시간을 계산하는 도메인 로직을 설계하고 구현하는 것입니다.

## 개발 방향
- 백엔드 중심으로 개발한다.
- MVP부터 작게 시작하고 점진적으로 확장한다.
- 포트폴리오와 면접에서 설명 가능한 구조로 만든다.
- 혼자 유지보수 가능한 수준의 복잡도로 설계한다.
- DDD를 적용하되, 실용적인 수준에서 사용한다.

## 기술 스택
- Java 17
- Spring Boot 3
- Spring Web
- Spring Data JPA
- H2 Database
- Validation
- 프론트엔드는 최소 수준으로 구성한다.
- 외부 API는 Kakao Local API / 길찾기 API를 사용한다.

## MVP 범위
- 출발지 입력
- 도착지 입력
- 목표 도착 시각 입력
- 도착 엄수 여부 입력
- 허용 지연 시간 입력
- 예상 경로 이동 시간 조회
- 추천 출발 시간 계산
- 추천 결과 저장
- 추천 결과 조회

## 이번 단계에서 제외할 범위
- 로그인 / 회원가입
- 복잡한 프론트엔드
- 실시간 교통 시각화
- 대중교통 / 도보 / 자전거 비교
- 알림 기능
- 캘린더 연동
- 과도한 마이크로서비스식 분리
- 필요 이상의 인프라 기술 도입

## 핵심 도메인
이 프로젝트의 핵심 도메인은
"도착 조건을 기반으로 추천 출발 시간을 계산하는 문제"이다.

즉, 외부 지도 API 자체가 핵심이 아니라
사용자의 도착 조건을 해석하고
이동 시간과 여유 시간을 반영하여
추천 출발 시간을 도출하는 로직이 핵심이다.

## 핵심 도메인 개념
- 도착 정책 (ArrivalPolicy)
- 예상 이동 시간 (TravelDuration)
- 여유 시간 정책 (BufferPolicy)
- 추천 출발 결과 (DepartureRecommendation)
- 추천 이력 (RecommendationHistory)

## 포트폴리오 관점에서 강조할 내용
- 외부 API 연동 로직과 도메인 로직을 분리했다.
- 도착 제약조건을 도메인 모델로 표현했다.
- 단순 조회가 아니라 규칙 기반 추천 문제로 풀었다.
- MVP를 작게 시작하고 확장 가능한 구조로 설계했다.
